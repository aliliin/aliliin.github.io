Go-kratos 框架商城微服务实战之用户服务 (五)

这篇主要编写对 web 端提供 http 接口的服务，主要还是一些准备工作。写的不清晰的地方可看源码 , 欢迎大佬指教。

注：竖排 … 代码省略，为了保持文章的篇幅简洁，我会将一些不必要的代码使用竖排的 . 来代替，你在复制本文代码块的时候，切记不要将 . 也一同复制进去。

  1. shop http 服务项目的创建
  • new 一个新的 kratos 项目代码
  • 删除 shop/api/helloworld/v1 目录下的所有文件
  • 修改或删除 shop 项目中的 shop/api/helloworld/v1 目录为 shop/api/shop/v1
  • 复制 service/user/api/user/v1/user.proto 文件到新建的 shop/api/service/user/v1 目录下
  • 新建 shop/api/shop/v1/shop.proto 文件

这里提供对外访问的接口，会聚合从不同的服务之间获取数据，接口路由通过 Protobuf IDL 定义对应的 REST API 和 gRPC API，

参数校验使用 Validate 中间件，使用 proto-gen-validate 生成

在使用 validate 之前首先需要安装 proto-gen-validate。

  • shop 根目录执行 make api ,生成 *pb.go 文件

如果你觉得这样子设计项目目录不合理的话，你可以自己设计目录。只要能把服务之间关联起来可以了。

注： 这里的 name，consul 的服务注册与发现要用到

项目中引入了 consul、trace，配置文件的时候，需要把相关的配置设置好，service 的配置是 consul 用来服务发现的
